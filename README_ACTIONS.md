# 共有データ配信（GitHub Actions）の導入手順

このフォルダの中身をリポジトリに追加すると、毎時サーバー側で天候・ニュースを取得して
`data/live.json` を自動更新し、**誰がいつどのブラウザで見ても同じ内容**になります。

## 手順（3分）
1. リポジトリの「Add file」→「Upload files」で、このフォルダの中身
   （`.github` フォルダ・`scripts` フォルダ・新しい `index.html`）をドラッグ&ドロップ
   ※ `.github` が隠しフォルダとして見えない場合は、zipを展開したフォルダ全体をドロップすればOK
2. 「Commit changes」をクリック
3. リポジトリ上部の「Actions」タブを開き、緑の「I understand… enable them」が出たらクリック
4. 左の「update-live-data」→右上「Run workflow」→「Run workflow」で初回を手動実行
   （以降は毎時7分に自動実行されます）

## 確認方法
- 数分後にリポジトリに `data/live.json` が生成されていれば成功
- アプリの混雑ゲージをクリック → 観光ニュース行の出典が「共有データ(GitHub)」になります

## 補足
- Actionsが止まっても、アプリは自動で従来のブラウザ取得に切り替わり動き続けます
- `data/days/日付.json` に毎時の天候が蓄積され、カレンダーの過去日グラフに
  雨の時間帯（薄青帯）と天気サマリーが表示されるようになります（導入日以降の日付から）
